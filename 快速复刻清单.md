# 速刷单词 App 快速复刻清单

这份文档是短版执行清单。  
适合你以后不想翻长文档时，直接照着做。

详版说明见：
- [APP实操手册.md](D:/15pro/Documents/Clodex/rapid-word-flutter/APP实操手册.md)

## A. 产品目标

先确认这 4 件事：
- 目标用户是谁
- 要解决什么问题
- 核心玩法是不是 `认识 / 不认识`
- 第一版是否只做：词书、刷词、错词本、同步

## B. 环境准备

1. 安装 Flutter
2. 安装 Git
3. 安装 Visual Studio 2022（Windows Desktop C++）
4. 安装 Chrome 或 Edge

检查 Flutter：

```powershell
flutter --version
```

## C. 进入项目

```powershell
cd D:\15pro\Documents\Clodex\rapid-word-flutter
flutter pub get
```

## D. 补 Windows 工程

如果缺 `windows/`：

```powershell
flutter create --platforms=windows --project-name rapid_word_flutter .
```

## E. 开启 Windows 开发者模式

如果遇到 symlink 报错：

```powershell
start ms-settings:developers
```

然后打开开发者模式。

## F. 跑 Windows 版

```powershell
flutter run -d windows
```

## G. 建 Supabase 项目

1. 打开 [Supabase Dashboard](https://supabase.com/dashboard)
2. 创建项目
3. 进入 `SQL Editor`
4. 执行：
   - [schema.sql](D:/15pro/Documents/Clodex/rapid-word-flutter/supabase/schema.sql)

## H. 拿 Supabase 配置

需要两个值：
- `Project URL`
- `Publishable key`

注意：
- URL 只写到 `.co`
- 不要带 `/rest/v1/`
- 只用 `Publishable key`

## I. 本地带 Supabase 运行

```powershell
flutter run -d windows --dart-define=SUPABASE_URL=https://your-project.supabase.co --dart-define=SUPABASE_ANON_KEY=your-publishable-key
```

## J. 补 Web 工程

如果缺 `web/`：

```powershell
flutter create --platforms=web .
```

## K. 本机浏览器跑 Web

```powershell
flutter run -d chrome --dart-define=SUPABASE_URL=https://your-project.supabase.co --dart-define=SUPABASE_ANON_KEY=your-publishable-key
```

## L. 上线前手机适配

确认：
- `web/index.html` 已补 `viewport`
- 词书页手机是单栏优先
- 刷词页单词强制单行

## M. 构建 Web

```powershell
flutter build web --release --dart-define=SUPABASE_URL=https://your-project.supabase.co --dart-define=SUPABASE_ANON_KEY=your-publishable-key
```

## N. 安装 Vercel

```powershell
npm i -g vercel
```

## O. 首次部署

```powershell
cd build\web
vercel
```

首次交互建议：
- `Set up and deploy?` 选 `Y`
- `Link to existing project?` 选 `N`
- 自己填项目名

## P. 正式部署

```powershell
cd build\web
vercel --prod
```

记住：
- 以后优先打开正式域名
- 不要长期依赖旧的临时预览链接

## Q. 每次更新线上版本

```powershell
cd D:\15pro\Documents\Clodex\rapid-word-flutter
flutter build web --release --dart-define=SUPABASE_URL=https://your-project.supabase.co --dart-define=SUPABASE_ANON_KEY=your-publishable-key
cd build\web
vercel --prod
```

## R. 更新后验证顺序

1. 登录页能不能打开
2. 首页能不能正常显示
3. 词书页是不是手机单栏
4. 刷词页单词是不是一行
5. 错词本能不能正常滚动
6. 点击 `认识 / 不认识` 是否流畅

## S. 常见报错最短处理

### `No pubspec.yaml file found`

```powershell
cd D:\15pro\Documents\Clodex\rapid-word-flutter
```

### `No Windows desktop project configured`

```powershell
flutter create --platforms=windows --project-name rapid_word_flutter .
```

### `Building with plugins requires symlink support`

```powershell
start ms-settings:developers
```

### `relation public.study_sessions does not exist`

去 Supabase 先执行：
- [schema.sql](D:/15pro/Documents/Clodex/rapid-word-flutter/supabase/schema.sql)

### `404: DEPLOYMENT_NOT_FOUND`

解决：
- 重新 `vercel --prod`
- 改用最新 `Production` 或正式域名
