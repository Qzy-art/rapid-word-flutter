## 项目目标

这是一款“快速刷词”应用，不是传统“背单词”。

核心体验：

- 用户只做 `认识 / 不认识` 的快速判断
- 不认识的词自动进入错词本
- 错词继续回刷
- 词书支持手动添加和批量导入
- 最终支持电脑和手机同步

## 为什么这个方向值得先做

这个项目最重要的不是功能堆得多，而是先打通闭环：

1. 词书能管理
2. 能刷词
3. 错词能回流
4. 进度能保存
5. 手机和电脑都能打开

只要这 5 个点打通，App 就已经具备可用价值。

## 这次真实开发顺序

实际推进顺序是：

1. 明确产品玩法
2. 做原型
3. 迁到 Flutter
4. 跑通 Windows
5. 接 Supabase
6. 跑通 Web
7. 上线到 Vercel
8. 修手机适配
9. 收性能

## 为什么不建议一开始就做手机包

如果一开始直接打 Android / iOS，常见会卡死在：

- 环境安装
- 模拟器或真机调试
- 登录回调
- 部署和分发

先走 `Windows + Web` 的好处：

- 成本低
- 验证快
- 更适合边改边试
